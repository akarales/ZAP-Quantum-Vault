## 3. Findings

> Severity scale: Critical / High / Medium / Low / Informational.
> **No Critical or High findings** in the audited additions.

### F-1 (Low) — `encrypted_secret_hex` is plaintext in memory; misleading name
**Location:** `@/home/anubix/Documents/CODE/ZAP_QUANTUM_VAULT/src-tauri/src/models/key.rs:37`
The field holds the raw 32-byte ML-DSA seed hex while in memory; it is only encrypted when
the keystore file is written (AES-GCM). The name implies it's always encrypted.
**Risk:** Clarity/maintenance; a future contributor could log/serialize it assuming it's
safe. The secret is wiped on drop and never crosses IPC, so no live exploit.
**Recommendation:** Rename to `secret_hex` (or `secret_seed_hex`) and/or add a doc-comment
clarifying the at-rest vs in-memory state. Keep the `Drop` zeroization.

### F-2 (Low) — `HybridSigner.secondary_secret` is a public field
**Location:** `@/home/anubix/Documents/CODE/ZAP_QUANTUM_VAULT/src-tauri/src/crypto/hybrid_signing.rs:49`
`pub secondary_secret: [u8; 32]` exposes the Ed25519 secret seed across module boundaries.
**Risk:** API hygiene; encourages accidental copying of secret material. Zeroized on drop.
**Recommendation:** Make the field private; expose only what callers need (already have
`secondary_public_key()`).

### F-3 (Low) — Local seed buffers not always `Zeroizing`
**Location:** `create_vault` / `restore_from_mnemonic`
(`@/home/anubix/Documents/CODE/ZAP_QUANTUM_VAULT/src-tauri/src/commands/vault.rs:333-337,361-367`)
The freshly derived `[u8; 64]` seed (and `phrase` String) live as plain stack values until
function end before being copied into `Zeroizing` state.
**Risk:** Slightly longer secret lifetime in unzeroed memory.
**Recommendation:** Wrap the local seed in `Zeroizing` immediately after derivation; the
phrase is necessarily returned for backup, but can be zeroized in the frontend store after
the backup gate (already cleared via `clearMnemonic`).

### F-4 (Low/Medium) — Replay nonce cache is in-memory only
**Location:** `@/home/anubix/Documents/CODE/ZAP_QUANTUM_VAULT/src-tauri/src/commands/airgap.rs:28-34`
`SeenNonces` resets on app restart, so an envelope could be replayed after a restart *within
its 300 s freshness window*.
**Risk:** Narrow (requires restart within 5 minutes + captured envelope). The freshness
window bounds it.
**Recommendation:** Optionally persist consumed nonces (or last-seen timestamp per signer)
to disk for defense-in-depth, or shorten the window for high-value transfer types.

### F-5 (Low) — AES-GCM uses no associated data (AAD)
**Location:** `@/home/anubix/Documents/CODE/ZAP_QUANTUM_VAULT/src-tauri/src/crypto/encryption.rs:34-48`
`encrypt_vault` binds no context (file name, record type, version) into the AEAD.
**Risk:** Low — a ciphertext from one context (e.g., verifier) can't be swapped into another
in practice because keys/nonces differ, but binding context is defense-in-depth.
**Recommendation:** Add a domain string as AAD (e.g., `"verifier"`, `"master_seed"`,
`"keystore"`) so a record can't be relocated even under key reuse mistakes.

### F-6 (Informational) — 1 GiB Argon2 may fail on low-memory machines
**Location:** `crypto/kdf.rs:20`
**Risk:** Usability on constrained hardware (the derivation allocates ~1 GiB).
**Recommendation:** Detect available RAM at create time and pick the strongest profile that
fits; persist it per-vault (the versioned-params machinery already supports this).

### F-7 (Informational) — `sign()` uses `unwrap()` after a size check
**Location:** `@/home/anubix/Documents/CODE/ZAP_QUANTUM_VAULT/src-tauri/src/crypto/mldsa87.rs:137`
Safe today (guarded by a length check) but `map_err` would be defense-in-depth. Carried over
from the prior audit.

### F-8 (Informational) — `KdfParams::default()` (high) diverges from `VaultState` serde defaults (legacy)
**Location:** `crypto/kdf.rs:62-66` vs `models/vault.rs:20-28`
`KdfParams::default()` returns the **high** profile, while the `VaultState` serde field
defaults return the **legacy** profile (intentionally, so a pre-KDF-block `vault.json`
re-derives with the parameters it was actually created under). This is correct but subtle:
a reader could wrongly assume `Default` means high everywhere.
**Risk:** None functionally (verified by `e2e_vault_state_serde_defaults_are_legacy_profile`
and `e2e_vault_state_high_kdf_profile_roundtrips`); footgun only.
**Recommendation:** Add a doc-comment on both spots cross-referencing each other.
